# ADR-005: Создание проекта Lab Playwright Expert

**Дата:** 2026-05-17
**Статус:** Принято
**Автор:** Штрейкбрехер

## Контекст

Лаборатория нуждается в систематизированной экспертизе по Playwright. Существующие кейсы (кросспостинг Hype Pilot, мониторинг сайтов) используют разрозненные скрипты без общей базы.

## Решение

Создан проект `lab-playwright-expert` — монорепозиторий с базовой библиотекой и скриптами.

### Архитектура

```
src/lab_playwright_kit/    — переиспользуемые модули
  browser.py               — управление браузером (async)
  stealth.py               — антидетект (webdriver, plugins, WebGL, etc.)
  parser.py                — парсинг DOM, извлечение данных
  screenshot.py            — скриншоты, PDF, visual diff
  network.py               — перехват запросов, моки
  llm_parse.py             — LLM-парсинг (браузер + модель)
scripts/                   — конкретные кейсы
  crosspost.py             — кросспостинг
  site_monitor.py          — визуальный мониторинг
tests/                     — 18 тестов, все зелёные
```

### Технологии
- **Python 3.10+** — единообразие со стеком лаборатории
- **Playwright 1.59** — async API, Chromium
- **loguru** — логирование
- **pydantic** — валидация конфигураций
- **pytest + pytest-asyncio** — тестирование

## Последствия

### Плюсы
- Единая база для всех Playwright-задач
- Антидетект из коробки
- LLM-парсинг как отдельный модуль
- Тесты покрывают основные сценарии

### Минусы
- Поддержка stealth-скриптов требует обновления при изменениях в антибот-системах
- LLM-парсинг зависит от доступности API
