# ADR-002: Шифрование сессий с Fernet

## Статус
Принято

## Контекст
Session Manager хранит cookies, localStorage, sessionStorage на диске. Эти данные могут содержать чувствительную информацию (токены, пароли). Необходимо защитить их от несанкционированного доступа.

## Решение
Используется Fernet шифрование (симметричное, AES-128-CBC + HMAC-SHA256):

- Ключ генерируется автоматически если не предоставлен
- Данные шифруются перед записью на диск
- Метаданные хранятся отдельно (без cookies/storage)
- Поддержка TTL для автоматического истечения

## Последствия

**Плюсы:**
- Стандартное шифрование (cryptography.fernet)
- Простота использования
- Автоматическая генерация ключа

**Минусы:**
- Потеря ключа = потеря сессий
- Нет поддержки асимметричного шифрования
- Производительность — шифрование/дешифрование на каждую сессию

## Альтернативы

1. **Без шифрования** — быстро, но небезопасно
2. **AES-GCM** — сложнее в реализации
3. **HashiCorp Vault** — enterprise-решение, избыточно для текущих задач
